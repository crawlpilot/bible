# Development Processes & Workflows

Engineering process knowledge expected of a principal engineer: not just writing code, but shaping how your team ships software reliably.

## Sub-directories

| Folder | Contents |
|--------|----------|
| `git/` | Git cheatsheet (plumbing, reflog, bisect), branching strategies (TBD vs Gitflow vs GitHub Flow), deployment processes (canary, blue-green, rollback, GitOps, DORA metrics) |
| `processes/` | Incident management, on-call runbooks, post-mortem templates |
| `workflows/` | Git branching strategies, PR review standards, code ownership |
| `agile-scrum/` | Sprint ceremonies, estimation, planning poker, retros |
| `ci-cd/` | Pipeline design, deployment strategies (blue-green, canary, feature flags) |
| `best-practices/` | Code review standards, Java best practices, REST API design, observability/monitoring, logging, configuration management, engineering standards (OWASP, incident management, production readiness checklist) |

## Key Topics for Principal Engineer Bar

- Designing the **deployment pipeline** for a 50-team engineering org
- Setting **SLO/SLI/SLA** targets and how to monitor and enforce them
- Running effective **post-mortems** that drive systemic change
- **Feature flag** strategies for safe rollouts
- **Observability** as a first-class engineering concern
