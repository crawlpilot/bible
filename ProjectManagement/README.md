# Technical Project Management

Principal engineers own delivery, not just architecture. This section covers the PM skills expected at the principal level.

## Sub-directories

| Folder | Contents |
|--------|----------|
| `frameworks/` | OKRs (goal-setting, grading, anti-patterns), V2MOM (Salesforce annual planning), RICE prioritization, Eisenhower matrix |
| `estimation/` | T-shirt sizing, story points, PERT estimation, complexity mapping, cross-team estimation, estimation anti-patterns |
| `metrics/` | Engineering KPIs (DORA, reliability, quality, security), business alignment, tracking & accountability, team health metrics |
| `roadmaps/` | Roadmap fundamentals, platform/infra sample, migration sample, tech debt roadmap, audience-specific communication formats |
| `stakeholder/` | Executive communication templates, steering committee updates |

## Principal Engineer PM Expectations

- Own the **technical roadmap** for a platform or domain
- Translate technical debt into **business risk** for executive audiences
- Drive **multi-team alignment** through influence, not authority
- Create and maintain **engineering strategy documents**
- Manage **technical dependencies** across org boundaries
- Balance **short-term delivery** with **long-term architectural health**
